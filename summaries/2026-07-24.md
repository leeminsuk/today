## 읽을거리/추적할 점

- AREX는 constraint-wise 자기감사가 실제 open-ended 질의에서 오답 방향을 얼마나 잡아내는지, autonomous context-update가 검증된 증거를 잘못 버리지 않는지 합성 밖 데이터로 확인할 것.
- "Same Dangerous Objective"의 compositional gap은 downstream 모델에 **상류 provenance를 서명된 형태로 강제 전달**하면 완화되는지, 3-agent 구조 밖의 실제 파이프라인에서도 재현되는지 후속 실험으로 분리할 것.
- ACM의 5 primitive는 특정 configuration 수치이므로, 조직 scope 위계가 커질 때 validated compaction의 linear 비용·fidelity가 유지되는지 다른 memory 벤치에서 재현할 것.
- KV-cache error certificate는 randomized eviction의 처리량 손실이 실제로 "no accuracy cost"에 그치는지, FlashAttention 계열·양자화·GQA와 결합했을 때 coverage 0.97이 유지되는지 실측할 것.
- TTEL은 실행 feedback이 약하거나 없는 도메인(장문 추론·개방형 생성)에서 token 단위 error localization의 정확도가 얼마나 떨어지는지, prefix 재사용이 오히려 오류를 고착시키는 경우가 있는지 볼 것.
- Kimi K3/Fable 의혹은 재무부·Moonshot의 공식 입장과 기술적 증거(출력 유사도, 학습 로그, 수출통제 조사)가 나오기 전까지 정치적 주장과 사실을 계속 분리해 기록할 것 — 오늘 전문가 반박으로 어제의 "미확인 주장" 판단이 강화됐다.
- 공격형 보안 가드레일 논쟁은 OpenAI Trusted Access·Anthropic Cyber Verification의 실제 승인 기준·이탈 규모가 공개되는지, "책임 있는 접근" 설계가 오남용 문책과 함께 작동하는지 추적할 것.
- AMD Helios·Etched는 벤더/회사 발표 수치이므로, 독립 벤치마크·실제 출하 시점·전력효율·수율이 확인될 때까지 "주장" 단계로 다룰 것.
